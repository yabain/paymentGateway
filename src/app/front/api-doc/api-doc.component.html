Documentation API

Grâce à digiKUNTZ Payments, vous bénéficiez d’un écosystème complet de solutions de paiement vous permettant de créer et gérer des liens de paiement directs, des plans d’abonnement, des collectes de fonds, ainsi que la vente de produits et services numériques ou physiques.

Domain: https://app.digikuntz.com Base URL: /dev Format: JSON

Introduction

Au-delà de ses fonctionnalités, digiKUNTZ Payments met à votre disposition une API de paiement robuste et sécurisée, vous offrant la possibilité de connecter votre compte digiKUNTZ Payments à vos propres applications tierces, sites web ou systèmes métiers. Cette intégration vous permet d’initier des transactions, d’encaisser des paiements et de centraliser l’ensemble de vos flux financiers sur un seul et unique compte.

Conçue pour répondre aux besoins des développeurs et des entreprises, l’API digiKUNTZ Payments offre une grande flexibilité d’intégration, une gestion simplifiée des paiements, ainsi qu’un haut niveau de fiabilité. Elle s’adapte aussi bien aux projets de petite envergure qu’aux applications à fort volume de transactions, tout en garantissant une expérience de paiement fluide pour vos utilisateurs.

L’API digiKUNTZ Payments permet aux développeurs et aux entreprises d’intégrer de manière simple, sécurisée et flexible les services de paiement de la plateforme digiKUNTZ au sein de leurs applications, sites web ou systèmes métiers.

Cette API expose un ensemble de points d’entrée permettant d’initier des paiements (pay-in), de suivre l’état des transactions, de consulter les soldes, ainsi que de gérer les clés d’accès nécessaires à l’authentification des requêtes. Elle est conçue pour centraliser l’ensemble de vos flux financiers sur un compte unique tout en garantissant la traçabilité, la fiabilité et la sécurité des opérations.

La documentation présente les différents endpoints disponibles, leurs méthodes HTTP, les paramètres requis, les formats de requêtes et les mécanismes d’authentification associés. Les échanges avec l’API s’effectuent au format JSON et reposent soit sur une authentification par jeton JWT, soit sur l’utilisation d’une paire userId et secretKey, selon le type d’endpoint.

Que vous développiez une application de paiement ponctuel, un système d’abonnement ou une plateforme à fort volume de transactions, l’API digiKUNTZ Payments s’adapte à vos besoins et vous offre une base fiable pour construire des expériences de paiement performantes et évolutives..


Étapes d’obtention des clés API
1 Création d’un compte de type Organisation

Seuls les comptes de type Organisation sont éligibles à l’obtention des API de paiement. Ce type de compte permet d’accéder aux fonctionnalités avancées nécessaires à l’intégration et à l’exploitation des services de paiement via API.

2 Vérification d’identité (KYC)

Vous devez compléter la procédure de vérification de vos informations en vous rendant dans Profil > Certifier mon identité.
À défaut, vous pouvez transmettre les éléments requis par email à l’adresse contact@yaba-in.com.

Les documents suivants sont obligatoires :

  1. • Votre identifiant de compte digiKUNTZ Payments (visible sous votre nom dans la page Profil)
  2. • Le logo de votre organisation
  3. • Une lettre d’engagement certifiant que l’API ne sera pas utilisée à des fins illégales
  4. • Une attestation de non-redevance datant de moins de trois mois
  5. • Votre registre de commerce

Une réponse vous sera communiquée dans un délai maximum de 72 heures après réception et validation des documents.

3 Activation de la clé API

Une fois votre compte validé, rendez-vous dans Profil > Espace Dev, puis activez votre clé API.

L’activation de cette fonctionnalité autorise des applications tierces à initier des transactions depuis votre compte digiKUNTZ Payments à l’aide de vos clés API. Il vous incombe de conserver ces clés de manière sécurisée et de ne jamais les exposer publiquement.

Transactions

POST /dev/transaction/:userId/:secretKey

Creer un paiement (payin) via API.

Params

  • userId (string)
  • secretKey (string)

Body

{
  "estimation": 100,
  "raisonForTransfer": "Test",
  "userEmail": "email@example.com",
  "userPhone": "691224472",
  "userCountry": "Cameroon",
  "senderName": "Kevin"
}

Response

{
  "id": "6977ef263980e24b64996062",
  "status": "pending",
  "transactionRef": "txPayin-1769467686449-6ce86896",
  "amount": 100,
  "paymentCurrency": "XAF",
  "paymentWithTaxe": 105,
  "paymentLink": "Lien de paiement"
}

Statuts: pending - success - closed - error
